This guide will walk you through installing Pulse, creating your first module fingerprint, and running the standalone CLI to watch a directory.

## Prerequisites

- Python 3.11 or higher
- An OpenAI or Anthropic API key (for the standalone LLM handler)

## 1. Installation

Install the package directly from GitHub (until it is published to PyPI):

```bash
pip install git+https://github.com/aniolowie/Macroa-Pulse.git
```

If you want to use the Anthropic API instead of OpenAI, install with the optional dependency:

```bash
pip install "macroa-pulse[anthropic] @ git+https://github.com/aniolowie/Macroa-Pulse.git"
```

## 2. Create a Fingerprint

A fingerprint tells Pulse what a specific module cares about. Create a file named `pdf_watcher.json` in your current directory:

```json
{
  "module_id": "pdf_watcher",
  "name": "PDF Watcher",
  "version": "1.0",
  "question_template": "I noticed a new PDF at {location}. Should I read and summarize it?",
  "default_threshold": 0.65,
  "signal_priors": {
    "filesystem": {
      "watch_dirs": ["~/Downloads"],
      "relevant_extensions": [".pdf"]
    }
  }
}
```

This fingerprint tells Pulse to watch the `~/Downloads` directory, pay special attention to `.pdf` files, and ask the provided question when it decides to escalate.

## 3. Run the CLI

Export your API key so the built-in LLM handler can use it:

```bash
export OPENAI_API_KEY="sk-..."
# Or if using Anthropic:
# export ANTHROPIC_API_KEY="sk-ant-..."
```

Start the Pulse watcher:

```bash
macroa-pulse watch ~/Downloads --module pdf_watcher.json
```

You should see output indicating that the Retina is watching the directory and the module is registered.

## 4. Trigger an Escalation

Open a new terminal window and create a dummy PDF file in your Downloads folder:

```bash
touch ~/Downloads/test_document.pdf
```

Switch back to the terminal running Pulse. Within a few seconds, you should see:
1. The Retina detecting the file creation
2. The Limbic layer scoring the event
3. The Prefrontal layer forming the question
4. The LLM handler sending the question to the model and printing the response
